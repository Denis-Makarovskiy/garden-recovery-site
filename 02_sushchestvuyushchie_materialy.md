# Garden Recovery — существующие материалы

Источник: действующий сайт **gardenrecovery.me**.

Документ — чек-лист для команды: что забираем как есть, не переснимаем и не переписываем. Заказчик подтвердил: «вся информация на текущем сайте валидна». Задача исполнителей — пройтись по каждому пункту, выгрузить актуальную версию и положить в сборку нового сайта.

> **Рабочий принцип.** Если материал есть на текущем сайте и помечен ниже как «забираем» — он используется без переработки. Изменяется только обвязка (разметка, оформление, размещение в новой структуре).

## 1. Бренд и айдентика

| Элемент | Что забираем | Где использовать в новой структуре |
|---|---|---|
| Логотип | Текущий логотип в исходнике (SVG / AI). Если только PNG — запросить векторную версию. | Шапка, футер, заставка видео. |
| Палитра | Цвета с действующего сайта (HEX-значения). | Стайлгайд нового сайта; при необходимости — расширение акцентами. |

| Тон визуала | Стиль фотографий и видео текущего сайта — референс для новой съёмки. | Сохранить эстетику. |

**Действие:** выгрузить весь brand-kit одной папкой (logo / colors / fonts / sample imagery). Если чего-то нет в исходниках — запросить у заказчика.

## 2. Команда

**Забираем с действующего сайта по каждому специалисту:**
- Полное имя.
- Фотография в текущем виде (если качество позволяет — оставляем, если нет — переснимаем; см. файл `03_materialy_na_razrabotku.md`).
- Роль и зона компетенции.
- Регалии, образование, стаж — если опубликованы.

**Состав, который должен быть выгружен (по тексту брифа на структуру):**
1. Лена — главврач.
2. Денис — психолог.
3. Роман — основатель / вдохновитель проекта.
4. Влад — физиотерапевт.
5. Нутрициолог.
6. Шеф-повар.
7. Массажист.
8. Дерматолог.
9. Косметолог.

**Что проверить отдельно:**
- Согласие каждого специалиста на публикацию фото и видео — есть ли подписанные согласия. Если нет — собрать перед съёмкой.
- Единый формат имён — на действующем сайте может быть разнобой («Лена» vs «Елена [Фамилия]»). Решить и привести к одному стандарту.
- Регалии — должны быть подтверждаемы (дипломы, сертификаты на руках). На сайте указываем только то, что подтверждено документально.

## 3. Юридическая локация

**Забираем с действующего сайта:**
- Юридическое наименование организации.
- Страну и регион регистрации.
- Лицензии, если опубликованы.
- Реквизиты для футера.

**Где использовать в новом сайте:**

- Дисклеймеры в медицинских блоках (программы, этапы).
- Schema.org разметка (MedicalBusiness / HealthAndBeautyBusiness).

**Что проверить отдельно:**
- Соответствие действующих формулировок локальному законодательству (юрисдикция Адриатического региона). Перед публикацией — проход юриста по новому сайту.
- Cookie-баннер и текст согласия на обработку ПД — должен соответствовать GDPR (если хотя бы один EU-резидент в аудитории).

## 4. Контакты

**Забираем с действующего сайта:**
- Telegram (@-username).
- WhatsApp (номер).
- E-mail для общих и приватных обращений.
- Любые мессенджеры/каналы, которые публикуются сейчас.

**Где использовать:**
- Блок «Контакты».
- Форма консультации — параметр «удобный канал связи».
- Подтверждение после отправки формы — fallback на WhatsApp для срочных случаев.

**Что не используем:**
- Общий номер call-центра (если есть) — режет приватный тон. Только прямые мессенджеры.
- Формы с длинными опросниками (если есть) — заменяем на 3-полевую короткую.

## 5. Описания программ — текстовые источники

**Забираем с действующего сайта** (если на нём опубликованы):
- Описания услуг и программ — фактологическая часть (что входит, длительность, противопоказания).
- Описание IV-терапии — теперь интегрирована в Pulse Reset и в курсы инфузионной терапии в более длинных программах.
- Описание чекапов — теперь интегрированы в Pulse Reset и в диагностику на входе всех программ.
- Описание физиотерапии, массажа, косметологии, дерматологии.
- Информацию о питании.

**Что делаем с этим материалом:**
- Сохраняем фактологию.
- Переписываем тон и структуру под новый каркас (см. `01_struktura_sayta.md`, раздел 4).
- **Структура программ полностью изменилась.** Новая сетка — по длительности (Pulse Reset 3 / Standard Recovery 5 / Extended Recovery 7 / Garden Foundations 14 / Garden Sabbatical 30). Четыре клинические линии (Executive Recovery / Body Balance Reset / Human Reserve Program / Post-Surgery Recovery) — теперь подкатегория внутри Standard и Extended, не самостоятельные программы.
- **Языковая рамка обязательна.** Все формулировки — через язык ресурса, не через язык патологии. См. `04_kopirayt.md`, раздел 0 «Языковая рамка».
- Цены — не переносим. На новом сайте цены не публикуются.

## 6. Фотоматериалы

**Забираем с действующего сайта (если качество позволяет, ≥ 2000 px по длинной стороне):**
- Фото территории и фасада.
- Фото интерьеров: жилые зоны, общие пространства, столовая, кабинеты специалистов.
- Фото деталей среды: ткани, посуда, природа, виды на побережье.
- Фото оборудования (диагностического, физиотерапевтического, IV-зоны и т.п.).
- Портреты специалистов команды.

**Что проверить:**
- Разрешение и качество — пригодны ли для современных Retina-экранов.
- Единый стиль обработки — если разнобой, переснимаем (см. `03_materialy_na_razrabotku.md`).
- Права на использование — все фото должны быть собственностью проекта или с действующей лицензией.

**Где использовать:**
- Фон главного экрана (постер для мобильной версии).
- Видео-фоны / статичные кадры в этапах процесса (раздел 3 нового сайта).
- Карточки программ (заставки в развороте).
- Карточки команды (портреты).
- Блок «Контакты» (фото территории).
- Тур по резиденции (если будет реализован — раздел 4 в `03_materialy_na_razrabotku.md`).
